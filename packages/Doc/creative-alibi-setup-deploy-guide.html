5 Configure Environment Variables

1
Copy .env.example
# Mac/Linux:
cp packages/backend/.env.example packages/backend/.env

# Windows PowerShell:
Copy-Item packages\backend\.env.example packages\backend\.env
2
Fill in packages/backend/.env
# IBM watsonx.ai (from Section 2)
WATSONX_API_KEY=your-ibm-cloud-api-key
WATSONX_PROJECT_ID=your-project-id
WATSONX_URL=https://us-south.ml.cloud.ibm.com
WATSONX_IAM_URL=https://iam.cloud.ibm.com/identity/token

# IBM Granite model
GRANITE_MODEL_ID=ibm/granite-3-3b-instruct
GRANITE_FALLBACK_MODEL_ID=ibm/granite-3-8b-instruct

# Hugging Face (from Section 3)
HUGGINGFACE_API_KEY=hf_your_token_here

# Server
PORT=3001
NODE_ENV=development
HOST=0.0.0.0

# Security keys — generate with the command in step 3
SESSION_ENCRYPTION_KEY=paste_64_char_hex_here
JWT_SECRET=paste_64_char_hex_here

# CORS — update after deploying frontend
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5173

# Limits
RATE_LIMIT_WINDOW_MS=900000
RATE_LIMIT_MAX_REQUESTS=100
GRANITE_MAX_TOKENS=300
GRANITE_TIMEOUT_MS=30000
DETECTOR_MAX_CALLS_PER_SESSION=1
3
Generate two secret keys — run this twice, use different outputs
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
# Paste output → SESSION_ENCRYPTION_KEY, then run again for JWT_SECRET
4
Create frontend .env files
echo "VITE_BACKEND_URL=http://localhost:3001" > packages/web-dashboard/.env
echo "VITE_BACKEND_URL=http://localhost:3001" > packages/word-plugin/.env

12 Word Plugin → GitHub Pages Free HTTPS static

Create .github/workflows/deploy-plugin.yml in your repo:

name: Deploy Word Plugin

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deploy.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm' }
      - run: npm install
      - run: npm run build --workspace=packages/shared
      - run: npm run build --workspace=packages/word-plugin
        env:
          VITE_BACKEND_URL: https://your-app.up.railway.app
      - uses: actions/upload-pages-artifact@v3
        with: { path: packages/word-plugin/dist }
      - id: deploy
        uses: actions/deploy-pages@v4
1
Repo → Settings → Pages → Source: GitHub Actions
2
Push the workflow file → auto-deploys in ~3 min
3
Plugin URL: https://YOUR_USERNAME.github.io/creative-alibi/

14 Troubleshooting

Run npm install --legacy-peer-deps instead.
Your WATSONX_API_KEY is wrong, or the Watson Machine Learning service is not associated with your project. Re-do Section 2 Step D. Make sure the WML instance is in us-south (Dallas).
The .env file must be at packages/backend/.env — not the root folder. Check for extra spaces around the = sign in the file.
The dev server is not running, or the HTTPS cert is not trusted.
1. Confirm Terminal 3 is running (npm run dev in packages/word-plugin).
2. Open https://localhost:3000 in your browser and click "Proceed".
3. Re-sideload the manifest in Word.
Insert → Add-ins → My Add-ins → right-click "Creative Alibi" → Remove. Restart Word. Re-upload the manifest.
Your Vercel or GitHub Pages URL is not in ALLOWED_ORIGINS on Railway. Add it and redeploy the backend.
In Vercel project Settings → General, confirm Root Directory is packages/web-dashboard. Check that VITE_BACKEND_URL is set in Environment Variables.
The model is cold-starting. Free tier puts models to sleep. Wait 30 seconds and retry — the code handles this gracefully and the report still generates with an "UNCERTAIN" signal.
